Le criticità sollevate richiedono tre interventi strutturali mirati: isolamento dello storage, hardening dei selettori interni e direttive di esclusione per i motori di rendering (stampa e canvas).

### Interventi Tecnici Implementati

1. **Isolamento Configurazione (Per-Site):**
La costante `SETTINGS_KEY` ora include l'`hostname` corrente. Questo genera configurazioni atomiche per ogni dominio, prevenendo la sovrascrittura incrociata dei parametri (es. posizione toolbar, auto-hide) tra siti diversi.
2. **Immunità DOM (Hardening Tools):**
Aggiunto un preset nativo `codeCornUI` in `EXCLUSION_PRESETS` che traccia tramite wild-card `[id^="__cc_"]` tutti gli elementi generati dallo script (sidebar, modal, toast, tooltip). Questo inibisce l'azione distruttiva di Thanos e le intercettazioni di Monitor/Sniffer sulla UI proprietaria.
3. **Soppressione Rendering Esterno (Screenshot & Print):**
* **Stampa:** Aggiunta una media query `@media print { [id^="__cc_"] { display: none !important; } }` nel CSS di base.
* **Screenshot:** Passato il parametro `ignoreElements` a `html2canvas` con una callback che scarta ricorsivamente qualsiasi nodo del DOM avente un ID appartenente al namespace `__cc_`.
